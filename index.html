DocGen_LLM

Outil de génération de documentation de code à l'aide de modèles de langage et de RAG.
  • Python
  • Docker
  • Serveurs de langage
  • LLM
  • RAG
  • documentation
  • mkdocs
  • Markdown
  • HTML
  • Open source
  • IA

Problématique principale : Générer une documentation de code de manière automatisée et agnostique au langage dans le cadre de projet manquants de documentation et/ou ayant une documentation peu claire

DocGen_LLM est un outil de génération de documentation de code qui utilise des modèles de langage en combinaison avec des serveurs de langage pour créer un contexte concis destiné à des modèles de faible capacité, afin de produire une documentation claire et concise pour l'ensemble d'une base de code. Il est conçu pour être indépendant du langage de programmation. L'outil s'intègre facilement avec des outils de lecture de documentation comme mkdocs. Il permet de générer une documentation à ajouter au code ou à utiliser dans une documentation externe au format Markdown ou HTML.

Caractéristiques :

  • Technologies utilisées : Python, Docker, Serveurs de langage, LLM, MKDocs
  • Objectifs : Automatiser la création de documentation de projet de code de manière agnostique au langage sur des projets manquants de documentation
  • Résultats : Amélioration de la lisibilité et de la compréhension du code, réduction du temps consacré à la documentation manuelle et augmentation de la qualité de la documentation générée

Serveur de Language TCL

Serveur de language TCL pour l'analyse de code.
  • TCL
  • Serveur de language
  • TCP
  • stdin/out

Problématique principale : Dans le cadre d'un besoin d'analyse d'un projet en TCL j'ai développé un serveur de language pour faciliter cette tâche et la rendre possible avec mon projet DocGen_LLM.

Ce serveur de language permet d'analyser le code TCL et d'en extraire des informations selon le standard LSP. Ce projet est une implémentation simple d'un serveur de langage TCL avec les capacités suivantes : extraction de symbols, recherche de définitions ainsi que références. Ce projet comprend une interface stdin/out et TCP.
