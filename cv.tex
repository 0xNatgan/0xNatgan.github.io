\section*{Projets}
\noindent\textbf{DocGen\_LLM} (\href{https://github.com/0xNatgan/DocGen_LLM}{github.com/0xNatgan/DocGen\_LLM})\\
Outil de génération de documentation de code à l'aide de modèles de langage et de RAG.

\noindent
\bubble{Python} \bubble{Docker} \bubble{Serveurs de langage} \bubble{LLM} \bubble{RAG} \bubble{documentation} \bubble{mkdocs} \bubble{Markdown} \bubble{HTML} \bubble{open source} \bubble{IA}

\begin{itemize}
    \item Problématique principale: Générer une documentation de code de manière automatisée et agnostique au langage dans le cadre de projet manquants de documentation et/ou ayant une documentation peu claire
    \item DocGen\_LLM est un outil de génération de documentation de code qui utilise des modèles de langage en lien avec des serveurs de langage pour générer une documentation claire et concise de l'ensemble d'une base de code d'un projet. Il est conçu pour être agnostique par rapport aux langages de programmation. L'outil s'intègre facilement dans les outils de lecture de documentation tels que mkdocs. L'outil permet de générer une documentation à ajouter au code ou à utiliser dans une documentation externe utilisant markdown ou HTML
\end{itemize}

\noindent\textbf{Serveur de Language TCL} (\href{https://github.com/0xNatgan/tcl-lsp}{github.com/0xNatgan/tcl-lsp})\\
Serveur de language TCL pour l'analyse de code.

\noindent
\bubble{TCL} \bubble{Serveur de language} \bubble{TCP} \bubble{stdin/out}

\begin{itemize}
    \item Problématique principale: Dans le cadre d'un besoin d'analyse d'un projet en TCL j'ai développé un serveur de language pour faciliter cette tâche et la rendre possible avec mon projet DocGen\_LLM.
    \item Ce serveur de language permet d'analyser le code TCL et d'en extraire des informations selon le standard LSP. ce projet n'est qu'une instance simple d'un serveur de langage TCL
\end{itemize}
